## [利用 Git 的 --author 标志阻止 GitHub 上的 AI 机器人垃圾信息](https://archestra.ai/blog/only-responsible-ai) ⭐️ 7.0/10

Archestra 的一篇博客文章描述了使用 Git 的 --author 标志来过滤其 GitHub 仓库中的 AI 机器人垃圾贡献，有效阻止了低质量的自动拉取请求。 这为面临日益增长的 AI 生成垃圾信息的开源维护者提供了一种实用且低开销的解决方案，有助于维护仓库质量和维护者的理智。 该方法依赖于将提交作者邮箱与贡献者的 GitHub 账户匹配；然而，正如社区评论所指出的，当贡献者更改邮箱地址时，这种方法可能会失效。

hackernews · ildari · May 18, 15:24 · [社区讨论](https://news.ycombinator.com/item?id=48181125)

**背景**: Git 的 --author 标志允许按作者姓名或邮箱过滤提交。GitHub 会为提交被合并的用户授予贡献者身份，这可能被 AI 机器人利用以获得更高权限。该技术帮助维护者在合并前识别并拒绝垃圾 PR。

## [语音 AI 系统易受隐藏音频攻击](https://spectrum.ieee.org/voice-ai-audio-attacks) ⭐️ 7.0/10
