## [Mozilla Deprecates Asm.js as WebAssembly Takes Over](https://spidermonkey.dev/blog/2026/05/20/saying-goodbye-to-asmjs.html) ⭐️ 8.0/10

Mozilla announced the deprecation of asm.js, a JavaScript subset that enabled near-native performance in the browser, as WebAssembly has fully replaced its role. This marks the end of an era for asm.js, a key precursor to WebAssembly that proved native-speed code on the web was possible, influencing modern web performance standards. Asm.js was a strict subset of JavaScript optimized for ahead-of-time compilation, used by tools like Emscripten to compile C/C++ code for the web. WebAssembly, first released in 2017, offers a more compact binary format and better performance, making asm.js obsolete.

hackernews · eqrion · May 20, 12:01 · [Discussion](https://news.ycombinator.com/item?id=48206340)

**Background**: Asm.js was introduced by Mozilla in 2013 as a way to run C/C++ code in the browser at near-native speeds, using a subset of JavaScript that engines could optimize aggressively. It was a response to Google's Native Client (NaCl) and Portable Native Client (PNaCl). WebAssembly, a joint effort by major browser vendors, became a W3C recommendation in 2019 and has since become the standard for high-performance web applications.
