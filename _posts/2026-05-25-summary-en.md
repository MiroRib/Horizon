This move enables shared infrastructure for quantum chip fabrication, reducing duplication of costly cleanrooms and accelerating the quantum computing industry. The foundry, Anderon, will use IBM's 300mm superconducting silicon process and is part of a $2 billion CHIPS quantum package that also funds competing modalities like trapped ion and photonic quantum computing. A pure-play foundry like Anderon offers fabrication services to multiple companies, similar to how TSMC operates for classical chips.

Mullvad has begun rolling out a mitigation against exit IP fingerprinting on its VPN servers, which prevents attackers from linking a user's activity across different servers by correlating exit IP addresses. This update significantly enhances user privacy by closing a loophole that could allow behavioral profiling across VPN sessions, setting a new standard for VPN anonymity. The mitigation ensures that no information is leaked about which exit address is used on another VPN server or by another user on the same server, effectively breaking the correlation between sessions.

hackernews · Cider9986 · May 25, 17:45 · [Discussion](https://news.ycombinator.com/item?id=48269580)

**Background**: Exit IP fingerprinting is a technique where an observer correlates a consistent VPN exit IP address across multiple services or sessions to build a behavioral profile. Unlike many VPNs that use a small set of shared exit IPs, Mullvad distributes users across many exit IPs per server, which ironically created a fingerprinting vector. This mitigation addresses that by randomizing or hiding the exit IP assignment.

## [C Extensions and Portability Across Compilers](https://lemon.rip/w/6-c-extensions-compilers/) ⭐️ 7.0/10

An article on lemon.rip discusses the challenges of writing portable C code across different compilers, focusing on common pitfalls with extensions like __attribute__ and offering practical solutions. This matters because C remains a foundational language in systems programming, and portability issues cause significant friction for developers targeting multiple platforms or compilers. Addressing these pitfalls can improve code reliability and reduce maintenance burden. The article highlights that many projects use #if !(defined __GNUC__ || defined __clang__ || defined __TINYC__) to ignore __attribute__, but this approach fails if __attribute__ is already defined by another header. It suggests more robust checks and references the Common Lisp ecosystem as a model for managing extensions.

hackernews · xngbuilds · May 25, 14:15 · [Discussion](https://news.ycombinator.com/item?id=48267126)

**Background**: C compilers often provide non-standard extensions like GCC's __attribute__ to enable features such as function attributes. These extensions improve functionality but break portability when code relies on them without fallbacks. Alternative compilers (e.g., TinyCC, MSVC) may not support these extensions, requiring conditional compilation or workarounds.
