## [Lathe: LLM-Powered Tutorials for Hands-On Learning](https://github.com/devenjarvis/lathe) ⭐️ 7.0/10

Lathe is a new open-source Go CLI that uses LLM agent skills (Claude Code, Cursor, Codex) to generate interactive, source-backed tutorials for any technical topic, which users then work through by manually typing code in a local web UI. This project offers a novel approach to using LLMs for education—focusing on active learning and retention rather than passive consumption, potentially filling gaps where high-quality human-written tutorials are scarce. Lathe generates tutorials with a table of contents, side-notes, exercises, and source citations; it also supports verifying the tutorial compiles and extending it with additional parts. The project is built with a 'vibecoded' approach and is currently best tested on macOS with Claude Code.

hackernews · devenjarvis · Jun 7, 11:16 · [Discussion](https://news.ycombinator.com/item?id=48433756)

**Background**: LLM-based coding assistants like Claude Code, Cursor, and Codex are typically used to generate code automatically, which can bypass the learning process for beginners. Lathe flips this paradigm by using the LLM to create structured learning materials that require the user to actively type and engage with the code, promoting deeper understanding. The project is inspired by the author's experience learning programming through PSP homebrew tutorials.

## [How Linear Achieves Speed via Client-Side Mutations](https://performance.dev/how-is-linear-so-fast-a-technical-breakdown) ⭐️ 6.0/10

A technical analysis reveals that Linear achieves its perceived speed by performing client-side mutations and background syncing, updating the UI instantly before confirming with the server. This approach sets a performance benchmark for web applications, but the trade-off in reliability and user experience is debated, highlighting the tension between speed and data consistency. The technique involves optimistic updates where the UI assumes success and later reconciles with the server, which can lead to stale data or sync lags if the server fails or conflicts arise.

hackernews · howToTestFE · Jun 7, 19:01 · [Discussion](https://news.ycombinator.com/item?id=48437609)

**Background**: Client-side mutations update the local state immediately, while background syncing asynchronously sends changes to the server. This pattern, common in offline-first apps, prioritizes responsiveness over strict consistency. Linear uses this to make interactions feel instant.
