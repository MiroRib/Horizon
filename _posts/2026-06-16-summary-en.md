If JWTs are misused, applications risk security breaches, but the discussion also clarifies appropriate use cases like service-to-service communication. The debate centers on whether the convenience of JWTs outweighs their security trade-offs, especially for browser-based sessions where revocation is critical.

## [Slay the Spire 2 Uses Custom PRNG for Cross-Platform Seed Consistency](https://tck.mn/blog/correlated-randomness-sts2/) ⭐️ 7.0/10

Slay the Spire 2 implements a custom pseudo-random number generator (PRNG) to ensure that seeds produce identical results across all platforms, avoiding the inconsistencies caused by platform-dependent standard library implementations. This change guarantees that players can share seeds and experience the same game state regardless of platform, which is critical for community challenges and speedrunning. It also future-proofs the game against changes in standard library PRNG implementations that could break past seeds. The custom PRNG is implemented in C# within the Godot engine, replacing the platform-dependent System.Random. The article notes that seeds are passed through a 32-bit hash function, limiting the effective seed space to about 4 billion, which makes brute-force searching for unwinnable seeds feasible.

**Background**: In the original Slay the Spire, seeds differed between desktop and mobile versions because the C# standard library's PRNG implementation varied across platforms. A custom PRNG ensures deterministic behavior regardless of the underlying platform or future library updates. PRNGs are algorithms that generate sequences of numbers that appear random but are fully determined by an initial seed value.
