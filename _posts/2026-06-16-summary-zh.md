## [停止在浏览器会话中使用 JWT](https://gist.github.com/samsch/0d1f3d3b4745d778f78b230cf6061452) ⭐️ 8.0/10

一篇广泛分享的 Gist 文章认为，JSON Web Tokens（JWT）在基于浏览器的用户会话中本质上不安全，指出了撤销和令牌生命周期的问题。该文章链接了 Paragon Initiative Enterprises 的详细批评，并引发了广泛的社区讨论。 这场辩论影响了数百万 Web 应用程序处理身份验证和会话管理的方式。如果 JWT 被误用，应用程序将面临安全漏洞风险，但讨论也澄清了适当的用例，如服务间通信。 批评者指出，JWT 在没有黑名单或短过期时间的情况下难以撤销，并且许多实现使用弱算法或在载荷中暴露敏感数据。支持者认为，使用短生命周期令牌（例如 5-15 分钟）配合刷新令牌可以缓解这些问题。

hackernews · dzonga · Jun 16, 16:49 · [社区讨论](https://news.ycombinator.com/item?id=48558147)

**背景**: JSON Web Tokens（JWT）是一种紧凑、URL 安全的令牌格式，用于身份验证和信息交换。它们常用于 Web 应用程序中维护用户会话，但与传统的服务器端会话不同，JWT 是自包含的，没有额外基础设施就无法在服务器端失效。辩论的核心在于 JWT 的便利性是否值得其安全权衡，尤其是在撤销至关重要的浏览器会话中。
