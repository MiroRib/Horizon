## [RFC 10008 Introduces HTTP QUERY Method](https://www.rfc-editor.org/info/rfc10008/) ⭐️ 8.0/10

RFC 10008 officially defines the HTTP QUERY method as a safe and idempotent alternative to GET that allows a request body, addressing caching and interoperability issues that arise when using GET with a body or POST for queries. This new method provides a standardized way to perform complex queries with a request body while preserving safe and idempotent semantics, which can improve caching behavior and API design consistency across the web. The QUERY method is designed for cases where the query is too complex to fit in a URL, such as large JSON filters or binary inputs, and it requires the request body to be part of the cache key, which may lead to unbounded cache keys.

hackernews · schappim · Jun 17, 10:51 · [Discussion](https://news.ycombinator.com/item?id=48568502)

**Background**: HTTP has traditionally used GET for safe, idempotent requests but without a request body, and POST for unsafe requests with a body. Developers often work around this by sending a body with GET (which is non-standard) or using POST for queries, both causing caching and interoperability problems. The new QUERY method standardizes the safe, idempotent query with a body.
