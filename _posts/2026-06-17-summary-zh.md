## [RFC 10008 引入 HTTP QUERY 方法](https://www.rfc-editor.org/info/rfc10008/) ⭐️ 8.0/10

RFC 10008 正式定义了 HTTP QUERY 方法，作为一种安全且幂等的 GET 替代方案，允许携带请求体，解决了使用带体的 GET 或 POST 进行查询时出现的缓存和互操作性问题。 这种新方法提供了一种标准化的方式，在保持安全和幂等语义的同时，使用请求体执行复杂查询，从而改善缓存行为和 API 设计的一致性。 QUERY 方法适用于查询过于复杂而无法放入 URL 的场景，例如大型 JSON 过滤器或二进制输入，它要求将请求体作为缓存键的一部分，这可能导致无界的缓存键。

hackernews · schappim · Jun 17, 10:51 · [社区讨论](https://news.ycombinator.com/item?id=48568502)

**背景**: HTTP 传统上使用 GET 进行安全、幂等的请求但不带请求体，使用 POST 进行带体的不安全请求。开发者通常通过给 GET 发送体（非标准）或使用 POST 进行查询来绕过限制，这都导致了缓存和互操作性问题。新的 QUERY 方法标准化了带体的安全、幂等查询。
