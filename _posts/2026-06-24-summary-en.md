## [Nub: A Bun-like toolkit for Node.js via preload hooks](https://github.com/nubjs/nub) ⭐️ 8.0/10

Colin McDonnell released Nub, an all-in-one toolkit for Node.js that adds transpilation (via oxc), module resolution hooks, and polyfills for APIs like Worker and Temporal, all running on stock Node.js without replacing the runtime. Nub brings Bun-like developer experience to Node.js without forking the runtime, offering a lightweight alternative that leverages existing Node.js infrastructure. It could reduce fragmentation in the JavaScript tooling ecosystem by providing a unified, additive toolkit. Nub uses Node's --require preload hook to inject an oxc-powered transpiler (packaged as a Node-API addon) and registers a module resolution hook. It is purely additive, meaning code ultimately runs on Node's actual engine and standard library implementations.

hackernews · colinmcd · Jun 24, 14:14 · [Discussion](https://news.ycombinator.com/item?id=48660267)

**Background**: Bun is an all-in-one JavaScript runtime that includes a built-in transpiler, bundler, and package manager, but it replaces Node.js entirely. Nub takes a different approach by augmenting Node.js with similar features via hooks, allowing developers to keep their existing Node.js setup while gaining improved developer experience. The oxc transpiler is a high-performance JavaScript tool written in Rust, used by projects like Rolldown and Nuxt.
