## [Nub：通过预加载钩子为 Node.js 提供类 Bun 工具包](https://github.com/nubjs/nub) ⭐️ 8.0/10

Colin McDonnell 发布了 Nub，这是一个面向 Node.js 的一体化工具包，通过 oxc 添加转译、模块解析钩子以及 Worker 和 Temporal 等 API 的 polyfill，所有功能均在原生 Node.js 上运行，无需替换运行时。 Nub 在不分支运行时的前提下，为 Node.js 带来了类 Bun 的开发体验，提供了一种利用现有 Node.js 基础设施的轻量级替代方案。通过提供统一的、可叠加的工具包，它可能减少 JavaScript 工具链生态的碎片化。 Nub 使用 Node 的 --require 预加载钩子注入基于 oxc 的转译器（打包为 Node-API 插件），并注册模块解析钩子。它是纯附加的，代码最终在 Node 的实际引擎和标准库实现上运行。

hackernews · colinmcd · Jun 24, 14:14 · [社区讨论](https://news.ycombinator.com/item?id=48660267)

**背景**: Bun 是一个一体化 JavaScript 运行时，内置转译器、打包器和包管理器，但它完全取代了 Node.js。Nub 采用不同的方法，通过钩子为 Node.js 添加类似功能，使开发者能够保留现有的 Node.js 设置，同时获得更好的开发体验。oxc 转译器是一个用 Rust 编写的高性能 JavaScript 工具，被 Rolldown 和 Nuxt 等项目使用。

## [Rust 社区推动 crates.io 与 GitHub 解耦](https://infosec.exchange/@mttaggart/116806641273303255) ⭐️ 8.0/10

Rust 社区正在积极讨论并致力于将 crates.io 的包发布功能与 GitHub 解耦，最近已合并一项 RFC（rust-lang/rfcs#3963），相关实现工作已经开始。 这一变化对于 Rust 包生态系统的可靠性和去中心化至关重要，减少了单点故障，确保发布 crate 不依赖于任何单一互联网服务。 解耦工作复杂且主要由志愿者推动，crates.io 问题跟踪器上列出了具体任务的路线图，欢迎志愿者贡献。

hackernews · speckx · Jun 24, 19:40 · [社区讨论](https://news.ycombinator.com/item?id=48664733)

**背景**: crates.io 是 Rust 官方包注册中心，目前发布 crate 需要 GitHub 账户进行身份验证。这种依赖长期以来一直是可靠性和去中心化方面的担忧，因为它将核心基础设施组件与单一商业平台绑定。

**社区讨论**: 社区评论普遍认为解耦是可取的，但也承认工作的难度和志愿者驱动的性质。有人指出 Go 的包管理是一个可参考的模式，而另一些人则强调 Rust 不是一个公司，进展取决于资金和兴趣。
