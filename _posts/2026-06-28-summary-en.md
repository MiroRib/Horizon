## [Why Polish Diacritics Like 'ś' Are Blocked by Browser Shortcuts](https://aresluna.org/the-curious-case-of-the-disappearing-polish-s/) ⭐️ 7.0/10

A 2015 article explains that Polish diacritic characters such as 'ś' are often blocked in browsers because the AltGr key used to type them is misinterpreted as a Ctrl+Alt shortcut, triggering browser keyboard shortcuts instead of inserting the character. This issue affects Polish users and other users of European languages that rely on AltGr for diacritics, hindering text input in web applications and highlighting a broader problem with keyboard event handling in browsers. The article provides a JavaScript fix that intercepts keydown events and prevents default browser behavior for AltGr-modified keys, but notes that the fix may not work on all platforms (e.g., Alt+Cmd+S on Mac still gets blocked).

hackernews · colinprince · Jun 28, 12:44 · [Discussion](https://news.ycombinator.com/item?id=48706814)

**Background**: The AltGr key (right Alt) on many keyboards is used to type additional characters like Polish diacritics (e.g., 'ś' via AltGr+S). However, browsers often treat AltGr as a combination of Ctrl+Alt, causing conflicts with browser shortcuts (e.g., Ctrl+S for save). This is a known issue in web development, as browsers do not expose a simple way to distinguish AltGr from Ctrl+Alt.
