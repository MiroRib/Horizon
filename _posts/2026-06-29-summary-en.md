## [WATaBoy: JIT-Compiling Game Boy to WASM Outperforms Native Interpreter](https://humphri.es/blog/WATaBoy/) ⭐️ 8.0/10

WATaBoy is a Game Boy emulator that dynamically recompiles SM83 instructions to WebAssembly at runtime, achieving faster performance than a native interpreter by leveraging the browser's JIT compiler. This approach enables high-performance emulation on platforms like iOS that restrict native JIT compilation, since browsers allow JIT for WebAssembly. It demonstrates a clever workaround that could inspire similar projects in other constrained environments. The emulator targets the Game Boy's SM83 CPU and compiles each instruction into equivalent WebAssembly code, which is then executed by the browser's WebAssembly runtime. Benchmarks show it outperforms a native interpreter, though Firefox is about 25% slower than Chrome and Safari.

hackernews · energeticbark · Jun 29, 15:02 · [Discussion](https://news.ycombinator.com/item?id=48720190)

**Background**: Emulators traditionally use interpretation or static recompilation. Interpretation is simple but slow due to per-instruction overhead. JIT compilation dynamically translates code to native machine code for faster execution, but is often restricted on iOS outside of web browsers. WebAssembly provides a portable target that browsers can JIT-compile, enabling this workaround.
